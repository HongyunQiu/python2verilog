🔬 Python 模拟 Verilog 行为的实现原理

一、核心思想：显式分离时序与组合逻辑

关键洞察：Verilog 的核心难点在于区分"什么时候计算"和"什么时候存储"。Python 模拟的关键是显式分离这两个阶段。

1.1 两个核心函数

PythonVerilog 对应作用
@combinational
def compute(self, scl, sda):		 always @(*) begin ... end 组合逻辑：根据当前输入和寄存器状态，计算下一个状态
@sequential
def clock(self):		 always @(posedge clk) begin ... end 时序逻辑：在时钟上升沿，将组合逻辑的结果写入寄存器

1.2 变量命名规范

前缀含义Verilog 对应示例
reg_ 寄存器（时序） reg reg_state → reg [1:0] reg_state;
wire_ 组合逻辑中间值 wire / reg (在always @(*)中) wire_next_state → reg [1:0] wire_next_state;

二、执行流程：step() 函数

def step(self, scl, sda):
    self.scl_in = scl          # 1. 保存输入
    self.sda_in = sda
    self.compute(scl, sda)     # 2. 组合逻辑：计算下一状态
    self.clock()               # 3. 时序逻辑：时钟上升沿，更新寄存器
    return self.wire_sda       # 4. 返回输出

三、组合逻辑（compute）详解

3.1 核心模式

@combinational
def compute(self, scl, sda):
    # 1. 计算组合逻辑信号
    self.wire_start = (not sda) and self.reg_sda_prev and scl and self.reg_scl_prev
    
    # 2. 根据当前状态(reg_state)和输入，计算下一状态(wire_next_state)
    if self.wire_start:
        self.wire_next_state = ADDR
        self.wire_next_bit_cnt = 0
    elif scl and (not self.reg_scl_prev):  # SCL上升沿
        if self.reg_state == ADDR:
            self.wire_next_shift = (self.reg_shift << 1) | sda
            self.wire_next_bit_cnt = self.reg_bit_cnt + 1
            ...
    
    # 3. 计算输出
    if self.reg_state == ACK:
        self.wire_sda = self.reg_ack
关键规则：compute() 中只能读取 reg_* 变量，只能写入 wire_* 变量。不能直接修改寄存器！

3.2 与 Verilog 的对应

Python (compute)Verilog (always @*)
self.wire_start = (not sda) and self.reg_sda_prev and scl and self.reg_scl_prev assign wire_start = (!sda_in) && reg_sda_prev && scl && reg_scl_prev;
self.wire_next_state = ADDR wire_next_state = ADDR;
self.wire_next_shift = (self.reg_shift << 1) | sda wire_next_shift = {reg_shift[6:0], sda_in};

四、时序逻辑（clock）详解

4.1 核心模式

@sequential
def clock(self):
    # 将所有 wire_* 值写入 reg_*
    self.reg_state = self.wire_next_state
    self.reg_shift = self.wire_next_shift
    self.reg_bit_cnt = self.wire_next_bit_cnt
    self.reg_sda_prev = self.sda_in
    self.reg_scl_prev = self.scl_in
    self.reg_is_write = self.wire_is_write
    ...
    # 数组更新
    if self.wire_next_registers:
        self.registers = self.wire_next_registers[:]

4.2 与 Verilog 的对应

Python (clock)Verilog (always @(posedge clk))
self.reg_state = self.wire_next_state reg_state <= wire_next_state;
self.reg_shift = self.wire_next_shift reg_shift <= wire_next_shift;
self.registers = self.wire_next_registers[:] reg_registers[reg_reg_ptr] <= wire_next_shift;

五、装饰器的作用

def combinational(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    wrapper._logic_type = "combinational"  # 标记逻辑类型
    return wrapper

def sequential(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    wrapper._logic_type = "sequential"     # 标记逻辑类型
    return wrapper
当前实现：装饰器只是标记，不改变行为。
未来扩展：可以用于自动代码生成、静态检查、文档生成等。

六、为什么这种方法有效？

优势说明
可审查性 Python 代码比 Verilog 更易读，人类可以审查逻辑正确性
可验证性 Golden Model（算法）vs Cycle Model（硬件行为）可以对比验证
渐进式开发 先写 Golden Model 验证算法，再写 Cycle Model 模拟硬件，最后生成 Verilog
降低认知负担 AI 只需理解 compute()/clock() 模式，不需要理解 Verilog 语法

七、完整示例：I2C 地址匹配

# Python Cycle Model
@combinational
def compute(self, scl, sda):
    if self.reg_state == ADDR:
        self.wire_next_shift = (self.reg_shift << 1) | sda  # 移位
        self.wire_next_bit_cnt = self.reg_bit_cnt + 1
        if self.reg_bit_cnt == 7:
            addr = self.wire_next_shift >> 1  # 使用 wire_next_shift！
            rw = self.wire_next_shift & 1
            if addr == self.slave_addr:
                self.wire_next_state = ACK
                self.wire_is_write = (rw == 0)

# 对应 Verilog
always @(*) begin
    if (reg_state == ADDR) begin
        wire_next_shift = {reg_shift[6:0], sda_in};
        wire_next_bit_cnt = reg_bit_cnt + 1'd1;
        if (reg_bit_cnt == 7) begin
            if ((wire_next_shift[7:1] == SLAVE_ADDR[6:0])) begin  // 使用 wire_next_shift！
                wire_next_state = ACK;
                wire_is_write = !sda_in;
            end
        end
    end
end

这就是 Python 模拟 Verilog 行为的核心原理：显式分离组合逻辑和时序逻辑，通过变量命名规范确保语义清晰，通过 step() 函数模拟时钟周期。
